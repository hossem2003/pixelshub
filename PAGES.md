# Pixels - Page Documentation

This document provides a comprehensive overview of all pages in the Pixels educational platform, their purposes, and key features.

---

## Core Pages

### 1. **index.html** - Home Page
**Purpose:** Landing page and main entry point for the platform  
**Key Features:**
- Hero section with platform introduction
- Subject categories showcase (Computer Science, Math, Physics, Languages, etc.)
- Featured courses and popular resources
- Quick access to main features
- Statistics overview (courses available, active contributors)
- Call-to-action buttons for browsing and creating content

**Target Users:** All visitors (new and returning)

---

### 2. **browse.html** - Browse Resources
**Purpose:** Comprehensive search and filter interface for all educational content  
**Key Features:**
- Advanced filtering sidebar (by subject, difficulty, rating, date)
- Grid/list view of available courses and notes
- Search functionality
- Sorting options (most popular, recent, highest rated)
- Pagination for results
- Subject tags for quick filtering

**Target Users:** Students and learners looking for specific content

---

### 3. **course.html** - Course Viewer
**Purpose:** Display individual course content with version control  
**Key Features:**
- Table of contents sidebar with expandable chapters
- Main content area with formatted course material
- Version panel showing all revisions
- Version comparison tools
- Course metadata (author, last updated, ratings)
- Navigation buttons (previous/next section)
- Floating action buttons for quick access
- Code syntax highlighting
- Interactive examples

**Target Users:** Students consuming educational content

---

## Content Creation & Management

### 4. **create.html** - Create New Resource
**Purpose:** Interface for contributors to create new courses or notes  
**Key Features:**
- Rich text editor with formatting toolbar
- Subject and difficulty selection
- Tags and categorization
- Preview mode
- Draft saving
- File upload support
- Markdown/rich text support
- Template selection

**Target Users:** Educators and content creators

---

### 5. **improve.html** - Improve Existing Content
**Purpose:** Collaborative editing interface for suggesting improvements  
**Key Features:**
- Side-by-side editor (original vs. proposed changes)
- Diff visualization
- Impact metrics prediction
- Change history
- Comments and explanations for edits
- Undo/redo functionality
- Improvement suggestions workflow
- Approval status tracking

**Target Users:** Contributors improving existing content

---

### 6. **versions.html** - Version History
**Purpose:** Timeline view of all versions for a specific resource  
**Key Features:**
- Visual timeline with version markers
- Version comparison tools
- Rollback capabilities
- Author information for each version
- Change summaries
- Score/rating progression
- Download specific versions
- Fork version functionality

**Target Users:** Contributors and curators managing content evolution

---

### 7. **compare.html** - Version Comparison
**Purpose:** Side-by-side comparison of two different versions  
**Key Features:**
- Split-panel view
- Color-coded differences (additions, deletions, modifications)
- Version selector dropdowns
- Synchronous scrolling
- Diff legend
- Line-by-line change highlighting
- Merge capabilities
- Export comparison report

**Target Users:** Contributors reviewing changes between versions

## Key Features Across All Pages

### Consistent Elements
1. **Header Navigation** - Logo, main nav, search bar, theme toggle, user menu
2. **Mobile Menu** - Responsive hamburger menu for mobile devices
3. **Dark/Light Mode** - Theme toggle available on all pages
4. **User Avatar** - Dropdown menu with quick access to user features
5. **Search Functionality** - Global search available in header
6. **Responsive Design** - All pages optimized for desktop, tablet, and mobile
7. **Accessibility** - ARIA labels and keyboard navigation support
8. **Loading States** - Consistent feedback for user actions

### Design System
- **Color Scheme:** Pixel-art inspired with neon accents (purple, pink, cyan)
- **Typography:** Pixel font for headers, clean sans-serif for body
- **Icons:** Font Awesome 6.5.1 for consistent iconography
- **Animations:** Subtle neon glow effects and hover states
- **Grid System:** Flexible layouts with CSS Grid and Flexbox

## User Flows

### New User Flow
1. **Login** (login.html) → Register account
2. **Workflow** (workflow.html) → Learn how to use platform
3. **Browse** (browse.html) → Discover content
4. **Course** (course.html) → Consume learning material

### Content Creator Flow
1. **Create** (create.html) → Submit new course
2. **Profile** (profile.html) → Track contributions
3. **Notifications** (notifications.html) → Get feedback
4. **Improve** (improve.html) → Update content based on feedback

### Contributor Flow
1. **Browse** (browse.html) → Find content to improve
2. **Course** (course.html) → Review existing material
3. **Improve** (improve.html) → Submit improvements
4. **Versions** (versions.html) → Track version history
5. **Leaderboard** (leaderboard.html) → See ranking

## Technical Notes

- All pages use shared CSS from `styles/main.css` for consistent styling
- Each page has its own CSS file in `styles/` directory for page-specific styling
- JavaScript functionality in `scripts/main.js` handles theme toggle and mobile menu
- All pages are fully responsive with breakpoints at 768px and 480px
- UTF-8 encoding for proper character display
- Font Awesome CDN for icon library
- No external dependencies except Font Awesome
